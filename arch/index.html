go back home

1. download + boot media

  1. grab the freshest ISO from archlinux.org/download.
  2. flash it to a USB stick with whatever tool you like.
  3. plug it into your puter, smack the BIOS with a metal pipe, and boot from USB.
  4. once you land in the live environment, run this:
pacman -Sy archinstall && archinstall

2. core stuff

go through the menus and set your normal locale/user/timezone stuff. these are the big picks for a stable setup.

disk layout

  • use best-effort partition layout (giga important if you don't need some stupidly retarded sextuple boot).
  • use btrfs with copy-on-write as filesystem.
  • pick all drives you want wiped and used for arch.
  • optionally encrypt disks if you want the extra safety.

hardware + services

  • audio: pipewire.
  • bluetooth: enable it now if you might need it, doing this later is annoying.
  • desktop: KDE Plasma.
  • networking: NetworkManager with default backend.

GPU driver choice

  • AMD: ATI open-source.
  • NVIDIA: open kernel modules option.
  • Intel: sell your GPU and buy AMD.

3. run install + reboot

double check your settings one more time. smack install and go make strawberry tea while it cooks, then reboot.

4. after first boot

  1. open KDE system settings.
  2. set animation speed to about 3 ticks faster than default.
  3. scroll down to software update and switch it to manual + after reboot (important).
  4. open konsole and run:
curl -fsSL https://christitus.com/linux | sh

on the left, hit system setup then pick build prerequisites, gaming dependencies, and whatever else you want.

yeah this is convenient, but still be smart and check what scripts are doing when you can.

5. program install rules ( important !!! )

please actually follow these if you want your system to stay chill.

use pacman for:

browsers, CLI apps, and stuff that needs proper system/file permissions.

use flatpak for:

chat apps, vscodium, email clients, and basically anything electron flavored.

use AUR carefully for:

obscure tools when they are not elsewhere. steam-adjacent stuff is usually fine.

avoid from AUR:

drivers and low-level core bits. do that and you are asking for a dead computer.

6. rollback strategy

timeshift is your oh-no button. use it. before big updates or experiments, take snapshots so you can roll back and move on with life.

be very careful with the AUR, and make snapshots a habit.

that is pretty much it. enjoy the rabbit hole :3
